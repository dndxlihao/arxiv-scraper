# Contributing to arXiv Scraper

Thank you for your interest in contributing to arXiv Scraper! Here are some guidelines to help you get started.

## How to Contribute

### Reporting Bugs
1. Check if the bug has already been reported in the Issues section
2. If not, create a new issue with:
   - A clear title and description
   - Steps to reproduce the bug
   - Expected vs actual behavior
   - Environment details (OS, Python version, etc.)

### Suggesting Features
1. Check if the feature has already been suggested
2. Create a new issue with:
   - A clear description of the feature
   - Use cases and benefits
   - Any implementation ideas you have

### Submitting Code Changes
1. Fork the repository
2. Create a new branch for your feature/fix
3. Make your changes
4. Add or update tests if applicable
5. Update documentation if needed
6. Submit a Pull Request

## Development Setup

1. Clone your fork:
   ```bash
   git clone https://github.com/yourusername/arxiv-scraper.git
   cd arxiv-scraper
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Run tests:
   ```bash
   # Add tests as we develop them
   ```

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable and function names
- Add docstrings for public functions and classes
- Keep functions focused and small

## Pull Request Process

1. Ensure your code passes any existing tests
2. Update the README.md if needed
3. Update the CHANGELOG.md with your changes
4. The PR will be reviewed and merged if approved

## Questions?

Feel free to open an issue for any questions about contributing!
